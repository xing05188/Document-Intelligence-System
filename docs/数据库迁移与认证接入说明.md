# 数据库迁移与认证接入说明

## 前置条件

1. `.env` 中配置数据库连接并开启 `DB_ENABLED=true`。
2. 已安装依赖（包含 `psycopg` 与 `python-dotenv`）。

## 新增迁移文件

- `sql/002_auth_user_file_scope.sql`

该迁移会新增并补齐以下能力：

1. `users`（手机号账号）
2. `auth_sessions`（登录会话）
3. 会话/消息/文件的 `user_id` 归属
4. 统一文件元数据扩展（source/role/task_uuid 等）
5. 任务与文档资产的用户、会话维度索引

## 迁移与校验脚本

- `scripts/migrate_and_validate_db.py`

### 仅校验（不写库）

```powershell
python scripts/migrate_and_validate_db.py --check-only
```

### 执行迁移并校验

```powershell
python scripts/migrate_and_validate_db.py --apply
```

### 执行迁移并做写读冒烟校验（事务回滚）

```powershell
python scripts/migrate_and_validate_db.py --apply --with-seed-check
```

## 认证接口

后端新增认证路由：

- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/auth/me`
- `POST /api/auth/logout`

## 前端接入

前端新增并接入：

1. 登录注册页：`frontend/src/components/AuthPanel.vue`
2. 认证 API：`frontend/src/api/auth.js`
3. 自动带 Token：`frontend/src/api/client.js`
4. WebSocket Token：`frontend/src/api/messages.js`

如果需要强制登录访问，可在 `.env` 增加：

```env
AUTH_REQUIRE_LOGIN=true
AUTH_SECRET_KEY=请替换为长随机字符串
AUTH_ACCESS_TOKEN_TTL_MINUTES=10080
```
